# カメラ修理データ 類似事例検索システム 利用ガイド

アプリ名: RepairAnalysis（ウィンドウ上部の表示名「カメラ修理データ 類似事例検索」）
バージョン: 1.0.0
対象OS: Windows 10 / 11

> このガイドは、配布された実行ファイル（exe）を使う方向けの総合ドキュメントです。
> 「導入（セットアップ）」と「操作マニュアル」の両方を含みます。
> はじめて使う方は、まず「1. これは何をするソフトか」と「3. 初回セットアップ」を読んでください。

> 📌 **〔要確認〕と書かれた箇所** は、配布元での運用に合わせて確定が必要な部分です。
> 配布担当者が値を埋めてから配布してください。

---

## 1. これは何をするソフトか

修理コメントなどの修理データをもとに、「いま調べたい問い合わせ内容」に**似た過去の修理事例を探し出す**ためのソフトです。

大まかな流れは次のとおりです。

1. 調べたい内容（問い合わせ）を文章で入力する
2. ソフトが、その内容を分類するための「観点（タグの枠組み）」を自動で作る
3. 手元の修理データ一件ずつに、その観点でタグを付ける
4. 付いたタグをもとに、問い合わせに近い事例だけを絞り込む
5. 結果を Excel などで出力する

分類とタグ付けには、社内で用意された AI サービス（Dify）を利用します。そのため、利用には後述の API キーなどの設定が必要です。

## 2. 使う前に用意するもの

| 必要なもの | 説明 | 備考 |
|---|---|---|
| 配布された一式（フォルダ） | exe を含むフォルダ。zip で配布されます | 〔要確認〕配布形態 |
| Dify の API キー（2種類） | 「スキーマ生成用」「タグ付け用」の2つ | 〔要確認〕各自入力か、設定済み配布か |
| CA 証明書ファイル | 社内 Dify へ安全に接続するための証明書 | 通常はフォルダ内 `certs` に同梱。無い場合は Windows の証明書ストアを自動利用 〔要確認〕 |
| 修理データ | CSV ファイル、または SQL Server 上のデータ | 〔要確認〕主にどちら |
| （SQL利用時のみ）ODBC ドライバ | 「ODBC Driver 18 for SQL Server」 | 〔要確認〕各PCに導入済みか |

## 3. 初回セットアップ

### 3-1. ファイルの配置

1. 配布された zip を、デスクトップなど分かりやすい場所に展開（解凍）します。
2. 展開してできたフォルダは、**中身ごとそのまま**置いてください。フォルダ内の exe や周辺ファイルの位置関係が変わると動かなくなることがあります。
3. フォルダの中に `RepairAnalysis.exe` があります。これが本体です。よく使う場合は、この exe を右クリック →「ショートカットの作成」でデスクトップにショートカットを作ると便利です。

> ⚠️ exe 単体だけを別の場所にコピーしても動きません。必ずフォルダごと扱ってください。

### 3-2. CA 証明書について

社内 Dify に安全に接続するために、CA 証明書が必要です。
このソフトは証明書を次の順で自動的に探します。**多くの場合、利用者が特別な操作をする必要はありません。**

1. **証明書ファイルがあれば、それを使います。**
   通常はフォルダ内の `certs` フォルダに `dify_ca.pem` という名前で同梱されています。
2. **証明書ファイルが見つからない場合は、Windows に登録済みの証明書を自動的に使います。**
   会社のPCでは、社内の証明書があらかじめ Windows に登録されていることが多く、その場合はファイルが無くても接続できます。

つまり、`certs\dify_ca.pem` が入っていればそれで動き、入っていなくても Windows 側に社内証明書があれば動く、という二段構えになっています。

〔要確認〕 証明書ファイルを別途配布する運用の場合は、受け取ったファイルを
`（フォルダ）\certs\dify_ca.pem` の場所に置いてください。

> もし接続でエラーが出る場合は、「設定」タブの証明書テスト機能（証明書パス欄の近く）で、
> 「ファイルが読み込めた」「Windows証明書ストアを使う」「どちらも使えない」のどれかを確認できます。
> 「どちらも使えない」と出た場合は、配布担当者に連絡してください。

### 3-3. ソフトの起動

`RepairAnalysis.exe` をダブルクリックします。
ウィンドウ上部に「カメラ修理データ 類似事例検索」と表示され、横並びのタブが出れば起動成功です。

> はじめての起動時、Windows の警告（「WindowsによってPCが保護されました」等）が出ることがあります。
> 〔要確認〕この場合の対処（「詳細情報」→「実行」を押す等）の社内ルールを記載してください。

### 3-4. 設定の入力（最初に一度だけ）

一番左の「**設定**」タブを開き、次を入力します。入力後、画面下の「**設定を保存**」を押します。

- **APIキー（スキーマ生成）**: 〔要確認〕配布元から渡される値
- **APIキー（タグ付け）**: 〔要確認〕配布元から渡される値
- **ベースURL**: 接続先の Dify のURL 〔要確認〕（社内Difyかクラウドか）
- **CA証明書パス**: 通常は初期値（`certs/dify_ca.pem`）のままでOK。ファイルが無くても Windows の証明書で自動接続を試みます
- **（SQLを使う場合のみ）** Server / Database / User / Password / Driver

> 入力した内容は、お使いのPC内の設定ファイルに保存されます。
> 保存場所: `C:\Users\（あなたのユーザー名）\AppData\Roaming\repair-analysis\settings.json`
> 次回からは入力不要です（同じPCを使う限り保持されます）。
>
> ⚠️ この設定ファイルには、APIキーやパスワードが**そのままの文字で**保存されます。
> 共有PCでの利用や、設定ファイルの取り扱いには注意してください。

## 4. 基本的な使い方（操作の流れ）

タブは左から「設定 → 問い合わせ → スキーマ編集 → データ取得 → タグ付け実行 → 絞り込み → 出力」の順に並んでいます。
**基本はこの左から右の順番で進めれば1回の作業が完了します。**

```
問い合わせを書く → 観点を自動生成 →（必要なら観点を手直し）
  → データを読み込む → タグ付け → 絞り込み → 出力
```

### ステップ1.「問い合わせ」タブ — 調べたい内容を入力する

1. 「問い合わせ内容」の欄に、調べたい症状や事象を文章で入力します。
   例：「カメラの液晶内にゴミのような点が見える。撮影画像には写らない」
2. 「detail軸の最大数」を必要に応じて変えます（0〜10、初期値4）。これは後で作られる「補助的な分類の観点」の最大数です。多いほど細かく分類できますが、その分タグ付けの負荷が増えます。迷ったら初期値のままで構いません。
3. 「**スキーマ生成 (Dify 1回目)**」ボタンを押します。
4. しばらくすると、分類の枠組み（スキーマ）が自動生成されます。

### ステップ2.「スキーマ編集」タブ — 観点を確認・手直しする（任意）

自動生成された分類の観点が一覧で表示されます。

- **tier（種別）**: `core`（主分類・必ず1つ）と `detail`（補助分類）があります。
- 各観点の名前・候補などを、ダブルクリックで編集できます。
- 「detail軸を追加」で観点を足すこともできます。

そのままで問題なければ、何もせず次のタブへ進んで構いません。

### ステップ3.「データ取得」タブ — 修理データを読み込む

1. 「データソース」で **CSV** か **SQL** を選びます。
2. **CSVの場合**: 「参照...」でファイルを選び、続いて使う「マッピング」を選びます。
   - マッピングとは、CSV内の列（例：「お客様コメント」）を、ソフトが使う項目（ユーザーコメント等）に対応づける設定です。
   - 〔要確認〕共通マッピングを配布する場合は、その名前と選び方をここに追記してください。
3. 「**読み込み**」を押すとデータが読み込まれ、プレビューに表示されます。
4. 「**前処理実行 (バッチ分割)**」を押します。これでタグ付けの準備が整います。

> 💡 修理データは「ユーザーコメント」「修理コメント」のどちらか一方が入っていれば処理できます。
> 内部コメント（internal_1/2）は無くても構いません。

### ステップ4.「タグ付け実行」タブ — データにタグを付ける

1. 「**タグ付け開始**」を押します。
2. 件数が多い場合、「○○件をタグ付けしようとしています。トークンを多く消費します。続行しますか?」という確認が出ます。問題なければ「はい」を押します。
   - この警告が出る件数は、初期設定で **500件** です。〔要確認〕運用上の目安があれば追記してください。
3. 進捗バーが進み、完了すると結果がまとまります。
4. 一部の処理が失敗した場合は「失敗バッチ」に表示され、「**失敗バッチを再実行**」で再試行できます。

> タグ付けは件数に応じて時間がかかります。途中でウィンドウを閉じないでください。

### ステップ5.「絞り込み」タブ — 近い事例を抽出する

1. 「問い合わせタグの指定」で、探したい事例の条件（各観点の値）を指定します。
2. 「最低 overall_relevance」（関連度の下限、初期値0.30）や「上位 N 件」で、結果の範囲を調整します。
3. 「**絞り込み実行**」を押すと、条件に合う事例が一覧表示されます。
4. 行を選ぶと、「根拠詳細」でなぜその事例が該当したかを確認できます。

### ステップ6.「出力」タブ — 結果をファイルに保存する

1. 「ファイル名タグ」に、分かりやすい名前を付けられます（例：`EOS_R7_AF_lowtemp`）。
2. 「**保存実行**」を押すと、ファイルが出力されます。
3. 出力されるファイル:
   - **Excel（results.xlsx）**: 「tagged（全件）」「ranked（絞り込み）」の2シート入り。**通常はこれを開けばOK**です。
   - Parquet（全件・絞り込み）: 他システムやデータ分析用の形式
   - スキーマ・メタ情報（JSON）: 使った設定の記録
4. 出力先フォルダは「設定」タブで変更できます（初期値はソフト内の `output` フォルダ）。

> 💡 出力された Excel は、そのまま Tableau でも開けます（シートを指定して接続します）。

## 5. よくある質問・困ったとき

**Q. 起動しても何も起きない／すぐ閉じる**
フォルダの中身が揃っているか確認してください。exe だけを移動していないか、ファイルが一部消えていないかをチェックします。解決しない場合は zip から展開し直してください。

**Q.「スキーマ生成」や「タグ付け」でエラーになる**
多くは設定の問題です。「設定」タブで APIキー2種が正しく入っているか、ベースURLが正しいか、CA証明書の場所が合っているかを確認してください。社内ネットワークに接続されているかも確認します。

**Q. 証明書（CA証明書）に関するエラーが出る**
このソフトは、証明書ファイル（`certs\dify_ca.pem`）があればそれを使い、無ければ Windows に登録済みの証明書を自動的に使います。エラーが出る場合は次を確認してください。
1.「設定」タブの証明書テストで状態を確認する（「ファイルが読み込めた」「Windows証明書ストアを使う」のいずれかなら証明書はOK）。
2.「どちらも使えない」と表示される、または証明書関連のエラーが続く場合は、お使いのPCに社内証明書が登録されていない可能性があります。配布担当者に連絡してください。

**Q. タグ付けの一部が失敗した**
「タグ付け実行」タブの「失敗バッチを再実行」を押してください。一時的な通信の問題であれば、再実行で成功することがあります。

**Q. 件数の警告（トークン消費）を毎回出したくない／基準を変えたい**
「設定」タブの「タグ付け警告件数」で基準件数を変更できます。大きな値にすれば警告は出にくくなります。

**Q. 設定をやり直したい／前の人の設定が残っている**
「設定」タブで値を入れ直して「設定を保存」してください。
完全に消したい場合は、設定ファイル（`C:\Users\（ユーザー名）\AppData\Roaming\repair-analysis\settings.json`）を削除します。

**Q. 別のPCに移したら設定が消えた**
設定は各PCごとに保存されるため、新しいPCでは「設定」タブでの再入力が必要です。これは仕様です。

**Q. 修理データのうち、コメントが一部しか無いものがある**
「ユーザーコメント」か「修理コメント」のどちらかが入っていれば処理できます。両方とも空の行は、有効な情報が無いため結果が乏しくなります。

## 6. 用語ミニ辞典

| 用語 | 意味 |
|---|---|
| スキーマ | 修理データを分類するための「観点」の枠組み。問い合わせ内容から自動生成される |
| 軸（axis） | 分類の観点ひとつひとつ。core（主分類）と detail（補助分類）がある |
| タグ付け | 各修理データに、スキーマの観点で値（タグ）を付けること |
| overall_relevance | 問い合わせとの関連度の高さ。絞り込みの基準に使う |
| バッチ | タグ付けをまとめて処理する単位。データを小分けにして順に処理する |
| マッピング | CSVの列名を、ソフトが使う項目名に対応づける設定 |
| Dify | 分類・タグ付けに使う社内のAIサービス |
